# ID Convention

Five roles. Every column fits exactly one. Schema lives in `packages/appshore/db/prisma/schema/`.

## 1. Internal PK — operational entities

- Type: `Int @id @default(autoincrement())`
- Where: Project, Task, Invoice, User, Tenant — all transactional tables
- Rule: All FKs reference these. Never expose to URLs or AI tool inputs directly.

## 2. Internal PK — audit / event / log entities

- Type: `String @id` generated by a UUIDv7 helper (time-sortable)
- Where: event logs, webhook delivery logs, login events, Desk episodes/steps/approvals,
  agent invocation logs, plan/add-on event tables
- Rule: Time-ordered for write locality + chronological queries.
  Use UUIDv7, not UUIDv4, on append-heavy tables.

## 3. Public business identifier

- Type: `String`, sequence-derived with prefix (e.g. `PRJ-20260706-001`, `INV-2026-0001`)
- Where: `Project.projectNumber`, `Invoice.invoiceNumber`, etc.
- Rule: Human-readable, branded, shown in UI / PDF / email / AI tool calls.
  Generated via `CounterService.nextValue(tenantId, scope)` (atomic Postgres UPSERT).
  Never duplicated in a sibling redundant `*Id` column.

## 4. Tenant URL slug

- Type: `String`, unique, hand-picked
- Where: the Tenant's public identifier only.
- Rule: Used for URL/subdomain routing. All other tables FK to `Tenant.id` (Int),
  never to the slug.

## 5. Opaque token (credential, not identifier)

- Type: `String`, generated via `nanoid(22)` (preferred) or `randomBytes(16).toString('base64url')`
- Where: share-link tables, `ApiKey` tokens, password-reset tokens, session tokens
- Rule: Lives in a dedicated table with `expiresAt` / `revokedAt` / audit fields.
  Never the row's PK. Never embedded on the operational entity row.
  Never embeds the business number into the token value.

## Banned for new code

- CUID anywhere.
- UUIDv4 (random) on audit tables. Use UUIDv7.
- Storing tokens on the entity row itself.
- A redundant string ID column that duplicates a sibling `*Number`.
- FKs that target a non-`id` column (e.g. the tenant slug).
