# Introduction

Platform provides REST APIs for building on a multi-tenant application foundation — authentication, users and teams, API keys, webhooks, and AI assistant (MCP) integrations.

## What You Can Build

With Platform APIs, you can:

- **Server-to-server integrations** - Authenticate with API keys and call any endpoint your role allows
- **Event-driven workflows** - Receive HMAC-signed webhook events when things change
- **AI assistant integrations** - Connect Claude, ChatGPT, or any MCP client to your workspace
- **Internal tools and dashboards** - Query users, teams, and platform data over REST
- **Custom domain features** - The platform is a starter; add your own domain endpoints alongside these

## Core Capabilities

### Authentication
API keys for integrations, OAuth 2.1 for AI assistants and third-party apps, and JWT sessions for first-party UIs.

### Multi-tenancy
Every request is tenant-scoped. Data isolation is enforced server-side, so an API key can only ever see its own tenant.

### Webhooks
Subscribe HTTPS endpoints to platform events and receive signed payloads with automatic retries.

### AI / MCP
A built-in Model Context Protocol server exposes your tools to AI assistants, with OAuth scopes and human-in-the-loop confirmation for writes.

## API Architecture

Platform APIs follow RESTful conventions:
- JSON request/response format
- Standard HTTP methods (GET, POST, PUT, DELETE)
- Consistent error handling
- Bearer token authentication (API keys)

## Rate Limits

- **Staging keys**: 1,000 requests/hour
- **Production keys**: 10,000 requests/hour
