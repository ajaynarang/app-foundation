# Plan a Route

This is the core of Platform — give it a driver, a truck, and a set of stops. Platform gives you back a fully optimized, HOS-compliant route with rest stops and fuel stops inserted automatically.

## Prerequisites

- An API key ([get one here](/docs/keys))
- A driver and vehicle in your tenant (see [Manage Your Fleet](/docs/getting-started/manage-fleet) if you haven't created these yet)

Set your environment variables:

```bash
export APP_API_KEY="sk_staging_your_key_here"
export APP_BASE="https://api-staging.appshore.in/api/v1"
```

## The scenario

**Driver:** Mike Johnson — CDL holder, 2 hours already driven today, 4 hours on duty.

**Truck:** Freightliner Cascadia — 150-gallon tank, currently at 60% fuel, 6.2 MPG.

**Route:** Chicago warehouse → Indianapolis customer → Columbus terminal.

**The challenge:** Mike has limited remaining drive time today. Platform will figure out if he needs a rest stop and where to fuel up.

## Understanding the response

Platform returns a route plan broken into segments. Each segment is either a drive, a rest stop, a fuel stop, or time at a stop (dwell).

```json

  "planId": "rte_f8e7d6c5",
  "status": "PLANNED",
  "hosCompliant": true,
  "totalDistanceMiles": 362,
  "totalDurationHours": 9.5,
  "segments": [
    {
      "type": "DRIVE",
      "from": "Chicago, IL",
      "to": "Indianapolis, IN",
      "distanceMiles": 181,
      "durationMinutes": 180,
      "hosAfter": {
        "driveMinutesUsed": 300,
        "driveMinutesRemaining": 360,
        "onDutyMinutesUsed": 420,
        "onDutyMinutesRemaining": 420
      }
    },
    {
      "type": "STOP",
      "location": "Indianapolis, IN",
      "purpose": "DELIVERY",
      "dwellMinutes": 45

      "to": "Columbus, OH",
      "distanceMiles": 181,
      "durationMinutes": 170,
      "hosAfter": {
        "driveMinutesUsed": 170,
        "driveMinutesRemaining": 490,
        "onDutyMinutesUsed": 170,
        "onDutyMinutesRemaining": 670

      "type": "STOP",
      "location": "Columbus, OH",

    }

```

### What Platform figured out

1. **HOS validation** — Mike has 2 hours already driven. After the 3-hour drive to Indianapolis, he'll be at 5 hours. After dock time and driving toward Columbus, he'd exceed the 11-hour drive limit. Platform inserted a **10-hour rest stop** near Richmond, IN.

2. **Stop sequence** — Platform confirmed Chicago → Indianapolis → Columbus is optimal (no reordering needed for this route, but for complex multi-stop routes Platform will optimize the sequence).

3. **Compliance guarantee** — `hosCompliant: true` means every segment respects federal HOS rules. No violations.

## Activate the route

Once you're satisfied with the plan, activate it to start monitoring:

```bash
curl -X POST $APP_BASE/routes/rte_f8e7d6c5/activate \
  -H "Authorization: Bearer $APP_API_KEY"
```

Once activated, Platform monitors the route 24/7 and sends alerts if anything changes — driver delays, dock hold-ups, HOS warnings, weather events.

## Check for alerts

After activation, poll for alerts or set up [webhooks](/docs/webhooks) to receive them in real time:

```bash
curl "$APP_BASE/alerts?routePlanId=rte_f8e7d6c5&status=ACTIVE" \
  -H "Authorization: Bearer $APP_API_KEY"
```
