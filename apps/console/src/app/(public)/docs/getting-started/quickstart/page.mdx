# Quickstart

Get from zero to a working API call in under 5 minutes.

## 1. Get your API key

Go to **[API Keys](/docs/keys)** and sign in with your Platform account. Click **Generate Key**, give it a name, and copy the key. It starts with `sk_staging_`.

Store it as an environment variable:

```bash
export APP_API_KEY="sk_staging_your_key_here"
```

## 2. Make your first request

Let's fetch your own user profile to verify your key works:

```bash
curl https://api-staging.example.com/api/v1/auth/me \
  -H "Authorization: Bearer $APP_API_KEY"
```

You should get a `200 OK` response with a JSON object describing the authenticated principal.

## 3. List the users in your tenant

```bash
curl https://api-staging.example.com/api/v1/users \
  -H "Authorization: Bearer $APP_API_KEY"
```

Every request is automatically scoped to your tenant — you only ever see your own data.

## 4. Subscribe to events

Register a webhook endpoint to receive platform events (HMAC-signed, with automatic retries):

```bash
curl -X POST https://api-staging.example.com/api/v1/webhooks \
  -H "Authorization: Bearer $APP_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.example.com/webhooks/platform",
    "events": ["*"]
  }'
```

See [Webhooks](/docs/webhooks) for payload formats and signature verification.
