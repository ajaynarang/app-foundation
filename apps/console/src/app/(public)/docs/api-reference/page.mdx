# API Reference

Platform provides a REST API for authentication, users and teams, API keys, and webhooks.

## Base URL

```
https://api-staging.example.com/api/v1
```

## Core Endpoints

### Auth
Identify the current principal and manage sessions.

**Key endpoints:**
- `GET /api/v1/auth/me` - Get the authenticated user
- `POST /api/v1/auth/refresh` - Refresh an access token
- `POST /api/v1/auth/logout` - End the session

### API Keys
Create and manage server-to-server keys.

**Key endpoints:**
- `POST /api/v1/api-keys` - Create a key
- `GET /api/v1/api-keys` - List keys
- `POST /api/v1/api-keys/{id}/rotate` - Rotate a key
- `POST /api/v1/api-keys/{id}/revoke` - Revoke a key

### Health
Liveness and readiness probes (no auth required).

**Key endpoints:**
- `GET /api/v1/health/live` - Liveness check
- `GET /api/v1/health/ready` - Readiness check (DB + Redis)

[Explore everything in the playground →](/docs/api-playground)

## Response Format

All API responses follow this structure:

### Success (200 OK)
```json
{
  "data": {
    // Response data here
  }
}
```

### Error (4xx, 5xx)
```json
{
  "statusCode": 400,
  "message": "Validation failed",
  "error": "Bad Request"
}
```

## Rate Limits

- **Staging keys**: 1,000 requests/hour
- **Production keys**: 10,000 requests/hour

Rate limit headers:
- `X-RateLimit-Limit` - Total requests allowed
- `X-RateLimit-Remaining` - Requests remaining
- `X-RateLimit-Reset` - Reset timestamp

## Status Codes

| Code | Description |
|------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request |
| 401 | Unauthorized |
| 404 | Not Found |
| 429 | Rate Limit Exceeded |
| 500 | Server Error |
