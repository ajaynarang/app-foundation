# AI Integrations

Platform is **AI-native**. In addition to the built-in Assistant AI chat, you can connect Platform to your favorite AI assistants and use natural language to work with your workspace from anywhere.

## How It Works

Platform exposes a **Model Context Protocol (MCP)** server that gives AI assistants secure access to your workspace data. When you connect an AI client to Platform:

1. You authenticate with your Platform account (OAuth 2.1)
2. You choose which permissions (scopes) to grant
3. The AI client can then call Platform tools on your behalf

All actions respect your role permissions and tenant isolation -- the AI can only do what you can do in the Platform app.

## Available Tools

The starter ships with a deliberately small toolset — it is your extension point:

| Tool | What it does |
|------|--------------|
| `health-check` | Verify the connection and server status |
| `search-kb` | Semantic search over the tenant knowledge base |

Add your own domain tools by registering `@Tool` providers in the backend MCP tools module — they appear to connected assistants automatically.

<Callout type="info">
  Write tools require a confirmation step -- the AI will ask you to confirm before executing.
</Callout>

## Connection URLs

Your Platform server URL depends on your environment. All MCP clients connect to the **base URL** — the server handles OAuth discovery and MCP requests automatically.

| Environment | Server URL |
|-------------|-----------|
| **Production** | `https://api.example.com` |
| **Staging** | `https://api-staging.example.com` |
| **Local Development** | `http://localhost:8000` |

<Callout type="info">
  Use the base URL for all clients (Claude.ai, Claude Desktop, ChatGPT, etc.). The server handles endpoint discovery via `/.well-known/oauth-authorization-server` automatically.
</Callout>

## Permission Scopes

When connecting, you'll be asked to approve access scopes:

| Scope | Access Level |
|-------|-------------|
| `knowledge:read` | Search the tenant knowledge base |
| `documents:read` | View documents |
| `documents:write` | Create and update documents |
| `integrations:read` | View integration status |
| `integrations:write` | Manage integrations |
| `comms:send` | Send outbound communications |

Scopes you define for your own tools appear here as well.
