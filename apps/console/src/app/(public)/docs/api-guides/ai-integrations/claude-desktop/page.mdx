# Claude Desktop

Connect SALLY to the [Claude Desktop](https://claude.ai/download) app as an MCP Connector. The setup is the same as Claude.ai — paste a URL, sign in, and start managing your fleet.

<Callout type="info">
  Requires Claude Desktop version 0.7+ with MCP support enabled.
</Callout>

## Setup (2 minutes)

### Step 1: Get Your Connector URL

| Environment | Connector URL |
|-------------|--------------|
| **Production** | `https://api.trysally.com` |
| **Staging** | `https://sally-api.apps.appshore.in` |

### Step 2: Add the Connector in Claude Desktop

1. Open **Claude Desktop**
2. Click your **profile icon** (bottom-left) and select **Customize**
3. Navigate to **Connectors** in the left sidebar
4. Click the **+** button (top-right) to add a new connector
5. Paste the connector URL from Step 1
6. Click **Connect**

### Step 3: Authorize SALLY

1. Claude Desktop will redirect you to the **SALLY consent page**
2. Sign in with your **SALLY account** (same credentials you use in the app)
3. Review the permission scopes being requested
4. Click **Authorize** to grant access

Once authorized, you'll be redirected back to Claude Desktop.

### Step 4: Start Using Sally

Open a new chat and try these prompts:

```
"What loads are currently in transit?"

"Show me driver HOS status for the fleet"

"What's my compliance score?"

"Generate an invoice for load L-1001"

"What are the overdue invoices?"

"Plan a route from Houston to Memphis"
```

## Alternative: JSON Config File

If you prefer to configure MCP servers manually via the config file:

1. Open **Claude Desktop** → **Settings** → **Developer** → **Edit Config**
2. Add the SALLY server:

```json
{
  "mcpServers": {
    "sally": {
      "url": "https://api.trysally.com"
    }
  }
}
```

3. Save and restart Claude Desktop
4. Look for the **hammer icon** in the chat input to confirm it's connected

**Config file locations:**
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

## Multiple Environments

You can add both production and staging:

1. Add the production connector URL: `https://api.trysally.com`
2. Add the staging connector URL: `https://sally-api.apps.appshore.in`

Both will appear as separate connectors in your settings.

## Troubleshooting

### Connector not showing up

- Verify you're on Claude Desktop v0.7+ with MCP support
- Check that the URL is the **base URL** (no path suffix)
- Try restarting Claude Desktop

### "Connection refused" for local dev

- Ensure the SALLY backend is running: `pnpm backend:dev` (from repo root)
- Verify the port matches your `.env` configuration (default: 8000)

### OAuth loop / can't authorize

- Disconnect and reconnect the SALLY connector in settings
- Ensure `OAUTH_ISSUER` and `NEXT_PUBLIC_APP_URL` are set correctly in your SALLY backend
- Check browser console for errors on the consent page

### "No tools available"

- You may have denied scopes during authorization. Disconnect and reconnect, approving all scopes
- Check your SALLY role — Drivers see fewer tools than Dispatchers or Admins
