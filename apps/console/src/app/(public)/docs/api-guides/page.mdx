# API Guides

These guides help you integrate Platform's API into your systems — from authenticating requests to subscribing to real-time events and connecting AI assistants.

## Base URL

| Environment | Base URL |
|---|---|
| Development | `http://localhost:8000/api/v1/` |
| Staging | `https://api-staging.example.com/api/v1/` |
| Production | `https://api.example.com/api/v1/` |

All endpoints documented in these guides are relative to the base URL. For example, `GET /users` means `GET https://api.example.com/api/v1/users`.

## Authentication

Every request must include a Bearer token in the `Authorization` header:

```bash
curl -X GET https://api.example.com/api/v1/users \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

See [Getting Started > Authentication](/docs/getting-started/authentication) for instructions on obtaining your token.

## Response Format

All responses return JSON.

### Successful Responses

Successful requests return the resource or collection directly:

```json
{
  "id": "usr_abc123",
  "email": "jane@example.com",
  "createdAt": "2026-02-10T14:30:00Z"
}
```

List endpoints return an array with pagination metadata:

```json
{
  "data": [ ... ],
  "meta": {
    "page": 1,
    "limit": 20,
    "total": 47
  }
}
```

### Error Responses

Errors return a consistent structure:

```json
{
  "statusCode": 422,
  "message": "Validation failed: email is required",
  "error": "Unprocessable Entity"
}
```

Common status codes:

| Code | Meaning |
|---|---|
| `400` | Bad request -- malformed JSON or missing required fields |
| `401` | Unauthorized -- missing or invalid Bearer token |
| `403` | Forbidden -- valid token but insufficient permissions |
| `404` | Not found -- resource does not exist |
| `422` | Validation error -- request body failed validation |
| `429` | Rate limited -- too many requests, back off and retry |
| `500` | Server error -- unexpected failure, contact support |

## Pagination

List endpoints accept `page` and `limit` query parameters:

```
GET /api/v1/users?page=2&limit=50
```

| Parameter | Default | Maximum | Description |
|---|---|---|---|
| `page` | `1` | -- | Page number (1-indexed) |
| `limit` | `20` | `100` | Items per page |
